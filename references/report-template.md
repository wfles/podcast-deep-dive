# Report Template

Copy this structure for every podcast analysis. Fill each section based on transcript evidence.

---

## ─── 节目信息 ───
- **播客/栏目**：
- **标题**：
- **时长/形式**：（单口独白、双人对谈、嘉宾访谈、圆桌讨论等）

## ─── 内容概要 ───
（按播客实际推进节奏分板块，每板块包含以下三项：）

### [板块标题/议题]
- **核心立场**：简明扼要复述主播/嘉宾的观点。
- **论证/细节**：支撑观点的关键案例、数据或逻辑推演。
- **【金句摘录】**：保留原汁原味的有张力的表达，必须为转录原文。

（重复板块直至覆盖全部重要议题）

## ─── 分析研判 ───

### 🌟 亮点与增量
- 内容在知识、情感或思维框架上提供了哪些新东西？
- 主播/嘉宾在表达上有什么独到之处？
- 是否有罕见视角或一手信息？

### 🧐 观察与省思
- 是否存在逻辑跳跃、论据单一或视角局限？
- 哪些论点值得进一步推敲或反证？
- 是否有利益冲突或立场偏差未被声明？

## ─── 评论区高光 (Community Insights) ───
（捕捉比节目本身信息密度更高或视角更独特的反馈）

### 事实补充/反证
（评论区提供的专业背景、数据补充、或直接反驳节目论点的事实）

### 思维跃迁
（评论中提出的比主播更深一层、或跳出原有框架的见解）

### 舆论共鸣/冲突
（大众对本期内容最核心的反馈点是什么？争议集中在哪里？）

## ─── 总结 ───
- **情绪价值 vs 信息价值**：大致占比（如 30:70）
- **最核心的贡献**：一句话概括本期最有价值的东西
- **最大遗憾**：一句话概括本期最明显的缺失
- **适合场景**：什么人/什么情境下值得听这期
